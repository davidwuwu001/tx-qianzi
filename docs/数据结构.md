# 数据结构

本文档列出了腾讯电子签 API 中常用的核心数据结构。

## 1. Agent (代理相关信息)
用于集团主企业代子企业操作的场景。
| 名称 | 类型 | 必选 | 描述 |
| :--- | :--- | :--- | :--- |
| ProxyOrganizationId | String | 否 | 被代理机构在电子签平台的机构编号 |
| AppId | String | 是 | 渠道应用标识 |
| ProxyOperator | UserInfo | 否 | 代理操作人信息 |

## 2. Admin (企业超管信息)
| 名称 | 类型 | 描述 |
| :--- | :--- | :--- |
| Name | String | 超管姓名 |
| Mobile | String | 超管手机号（通常打码显示） |

## 3. FlowApproverInfo (签署方信息)
用于定义合同签署人的详细信息。
| 参数名称 | 类型 | 必选 | 描述 |
| :--- | :--- | :--- | :--- |
| ApproverType | Integer | 是 | 签署人类型：0-企业，1-个人，3-静默签 |
| ApproverName | String | 是 | 签署人姓名 |
| ApproverMobile | String | 是 | 签署人手机号 |
| OrganizationName | String | 否 | 企业名称（企业签署人必传） |
| SignComponents | Array | 否 | 签署控件列表 |

## 4. Component (填写/签署控件)
| 参数名称 | 类型 | 描述 |
| :--- | :--- | :--- |
| ComponentType | String | 控件类型：SIGN_SIGNATURE (签名), SIGN_SEAL (印章) 等 |
| ComponentPage | Integer | 控件所在页码 |
| ComponentPosX | Float | 控件横坐标 |
| ComponentPosY | Float | 控件纵坐标 |
| ComponentWidth | Float | 控件宽度 |
| ComponentHeight | Float | 控件高度 |

## 5. AdminChangeInvitationInfo (超管变更信息)
| 名称 | 类型 | 必选 | 描述 |
| :--- | :--- | :--- | :--- |
| ChangeAdminOrganizationId | String | 是 | 要变更的企业 Id |
| NewAdminName | String | 是 | 新超管姓名 |
| NewAdminMobile | String | 否 | 新超管手机号 |
| AuthFiles | Array | 是 | 授权书 Base64 格式 |
