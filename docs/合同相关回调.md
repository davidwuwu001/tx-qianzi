# 合同相关回调

当合同状态发生变化时，腾讯电子签会发送此类型的通知。

## 1. 消息类型
`MsgType`: `FlowStatusChange`

## 2. 核心状态码 (FlowCallbackStatus)
- `INIT`：合同创建
- `PART`：签署中（部分人已签）
- `ALL`：签署完成（所有人已签）
- `REJECT`：拒签
- `CANCEL`：撤销
- `OVERDUE`：已过期

## 3. 消息结构示例 (解密后)
```json
{
    "MsgId": "32位唯一标识",
    "MsgType": "FlowStatusChange",
    "MsgData": {
        "FlowId": "合同ID",
        "FlowName": "合同名称",
        "FlowStatus": 1,
        "FlowCallbackStatus": "ALL",
        "OperateTime": 1604910798
    }
}
```

## 4. 处理逻辑建议
1. 接收到 `ALL` 状态后，触发合同归档或下载流程。
2. 接收到 `REJECT` 或 `CANCEL` 状态后，更新业务系统的合同状态并通知相关人员。
