# 一码多签

一码多签允许通过一个二维码或链接，让多个人员扫码签署同一份合同。

## 1. 适用场景
- **现场签署**：如会议签到、现场入职协议。
- **批量签署**：针对不确定具体签署人的通用协议。

## 2. 核心接口
- **CreateMultiFlowSignQRCode**：创建一码多签签署码。
- **CancelMultiFlowSignQRCode**：取消一码多签签署码。

## 3. 核心逻辑
1. **创建模板**：先在控制台创建一个带签署控件的模板。
2. **生成码**：调用接口指定模板 ID，设置最大签署人数、有效期等。
3. **扫码签署**：签署人扫码后，系统会自动为其创建签署任务。

## 4. 关键限制
- 必须基于模板发起。
- 扫码人需在手机端完成实名认证后方可签署。
- 每一个签署人签署完成后，系统会生成独立的合同记录。
