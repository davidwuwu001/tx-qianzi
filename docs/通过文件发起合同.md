# 通过文件发起合同

直接使用现有的 PDF 文件发起签署流程。

## 一. 核心流程
1. **上传文件**：调用 `UploadFiles` 接口获取 `FileIds`。
2. **创建签署流程**：调用 `CreateFlowByFiles` 接口。

## 二. 操作步骤

### 步骤 1：上传合同原文件
- **接口**：`UploadFiles`
- **输入**：PDF 文件的 Base64 编码或文件流。
- **输出**：资源编号 `FileId`。

### 步骤 2：创建并发起签署流程
- **接口**：`CreateFlowByFiles`
- **关键参数**：
    - `FlowName`：合同名称。
    - `FileIds`：步骤 1 获取的资源编号。
    - `Approvers`：签署方列表（个人或企业）。
    - `Components`：填写控件（由发起方填写）。

## 三. 签署方配置
- **个人签署方**：需提供 `Name` 和 `Mobile`。
- **企业签署方**：需提供 `Name`、`Mobile` 和 `OrganizationName`。

## 四. 控件定位方式
1. **绝对定位**：指定 X, Y 坐标和页码。
2. **关键字定位**：搜索 PDF 中的文本关键字进行定位。
3. **表单域定位**：通过 PDF 表单域名进行定位。

## 五. 执行结果
接口调用成功后，将返回 `FlowId`，您可以根据此 ID 生成签署链接。
